Context-Free Grammar — Formal Definition

A CFG is a 4-tuple G = (V, Σ, R, S):

• V — finite set of non-terminal variables

• Σ — finite set of terminals (alphabet)

• R ⊆ V × (V ∪ Σ)* — production rules

• S ∈ V — start symbol

Derivation: w ∈ L(G) iff S ⟹* w. Each step replaces a non-terminal using one production rule. The language L(G) = {w ∈ Σ* | S ⟹* w}.

Example — aⁿbⁿ (n ≥ 1):
S → aSb | ab
Derives: ab, aabb, aaabbb, ...

Pushdown Automaton — Formal Definition

A PDA is a 7-tuple M = (Q, Σ, Γ, δ, q₀, Z₀, F):

• Q — finite set of states

• Σ — input alphabet

• Γ — stack alphabet

• δ: Q × (Σ ∪ {ε}) × Γ → P(Q × Γ*)

• q₀ ∈ Q — start state

• Z₀ ∈ Γ — initial stack symbol

• F ⊆ Q — accepting states

A PDA accepts by final state OR empty stack. Both acceptance modes are equivalent in power. Our converter uses empty-stack acceptance.

The Equivalence Theorem

Theorem (Chomsky, 1962): A language L is a Context-Free Language (CFL) if and only if there exists a Pushdown Automaton M such that L(M) = L.

Corollary: Every CFG can be converted to an equivalent PDA, and every PDA can be converted to an equivalent CFG. This bidirectional equivalence shows that CFGs and PDAs define exactly the same class of languages — the Context-Free Languages.

Intuition: A CFG describes the structure of strings (parse trees), while a PDA describes the process of recognizing them (using a stack to track nesting). The stack in a PDA mirrors the call stack of recursive descent parsing — each grammar variable corresponds to a stack frame.
